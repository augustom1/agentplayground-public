Features

Everything you need to run autonomous agents

🤖

Agent Teams

Organize Claude-powered agents into named teams with individual system prompts, models, temperature, and permission scopes. Seed pre-built teams like Dev Core, DevOps, and Command Center.

💬

Tool-Use Chat

Chat with Claude through a streaming interface that has direct access to your database. Claude can create tasks, query teams, log improvements, and generate new tools mid-conversation.

🗓️

Task Scheduling

Create one-off tasks or recurring cron jobs per team. View everything in a calendar UI. Cron jobs run via a secure external trigger endpoint protected by CRON_SECRET.

🧪

Agent Lab

A live playground for testing agent prompts against any team. Run history is persisted so you can compare results, tweak system prompts, and iterate quickly.

🔧

Tools Catalog

Every team has a library of Skills, CLI Functions, and logged Improvements. Claude can detect repeated patterns and generate new tools that get stored in the catalog automatically.

📊

Dashboard Widgets

Configurable stat, feed, calendar, and optimization widgets give you a live overview of team health, recent activity, tasks counts, and tool catalog growth.

🔐

Auth & Roles

NextAuth v5 with JWT strategy. Four role tiers: admin, builder, standard, readonly. Each maps to a distinct permission preset controlling what agents can read and write.

🐳

Docker Deployment

Single bash setup.sh on any VPS spins up the full stack: dashboard, PostgreSQL, Redis, Nginx, n8n, Ollama with auto-pulled local models, FileBrowser, and Portainer.

🧠

Local LLM Support

Ships with a custom Ollama image that auto-pulls qwen2.5:3b and qwen2.5:7b on first start. Route lightweight tasks to local models, fall back to Claude for complex ones.

Quick Start

Clone and run in minutes

You need Node.js 20+, Docker, and an Anthropic API key. Choose local dev or full VPS deployment.

1

Clone the repository

git clone https://github.com/agentplayground/agent-dashboard-ui-app.git
cd agent-dashboard-ui-app
2

Create your environment file

Copy the example and fill in the required secrets.

cp .env.example .env

Minimum required variables:

DATABASE_URL="postgresql://user:pass@localhost:5432/agentdb"
AUTH_SECRET="your-random-secret-32-chars"
ANTHROPIC_API_KEY="sk-ant-..."
NEXTAUTH_URL="http://localhost:3000"
3

Start the database (Docker)

docker compose up -d
4

Install dependencies and run migrations

npm install
npx prisma migrate dev
5

Start the dev server

npm run dev

Open http://localhost:3000/setup to create your admin account. The app redirects there automatically on first run.

🚀 Full VPS deployment

For production on a Linux VPS (Ubuntu 22+), a single bootstrap script handles everything — Docker install, Nginx, SSL, Ollama with local models, n8n, FileBrowser, and Portainer. 

# On your VPS — edit .env.local first, then:
bash setup.sh

DNS: point @ and * A-records to your VPS IP before running. See full deployment guide →

Seed pre-built agent teams

npm run seed:teams

Seeds five ready-to-use teams: Dev Core, DevOps & Infrastructure, Product & Design, Business & Growth, and Command Center (coordinator that routes to all others).
